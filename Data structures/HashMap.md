Ключи и значения в HashMap могут быть любых типов, в т.ч и null

Свойства (поля) класса HashMap:
- **table** — Массив типа **Entry[]**, который является хранилищем ссылок на списки (цепочки) значений;
- **loadFactor** — Коэффициент загрузки. Значение по умолчанию 0.75 является хорошим компромиссом между временем доступа и объемом хранимых данных;
- **threshold** — Предельное количество элементов, по достижении которого размер хэш-таблицы увеличивается вдвое. Рассчитывается по формуле **(capacity * loadFactor)**;
- **size** — Количество элементов HashMap-а;

##### Добавление элемента:

1) Ключ проверяется на null, если true - то вызывается метод putForNullKey(value). Внутри этого метода нет методов hash() и indexFor(), поэтому все элементы с ключом null помещаются в table[0]. 
2) Вычисляется хэш на основе ключа.
3) Определяется на основе хэша позиция в массиве, на которую будет помещен элемент.
4) Если в ячейке массива по определенной позиции есть цепочка (дерево с Java 8) элементов, то хэш и ключ нового элемента сравниваются с таковыми у элементов в ячейке, и при совпадении перезаписываются
5) Если нет, то добавится новый элемент в ячейку
6) В случае коллизии (совпадение хэшей разных объектов) элемент поместится в начало структуры в ячейке

HashMap имеет встроенные итераторы, такие, что вы можете получить список всех ключей **keySet()**, всех значений **values()** или же все пары ключ/значение **entrySet()**.
