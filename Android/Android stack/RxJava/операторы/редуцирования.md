##### skipWhile and takeWhile
take и skip работают с предопределенными индексами. Если вы хотите "обнаружить" точку отсечения по мере поступления значений, takeWhile и skipWhile будут использовать предикат. takeWhile принимает элементы, а функция предиката возвращает true
```java
Observable<T> 	takeWhile(Func1<? super T,java.lang.Boolean> predicate)
```
```java
Observable<Long> values = Observable.interval(100, TimeUnit.MILLISECONDS);

Subscription subscription = values
	.takeWhile(v -> v < 2)
	.subscribe(
	    v -> System.out.println(v),
	    e -> System.out.println("Error: " + e),
	    () -> System.out.println("Completed")
	);
```

```Java
Observable<Long> values = Observable.interval(100, TimeUnit.MILLISECONDS);

Subscription subscription = values
	.skipWhile(v -> v < 2)
	.subscribe(
	    v -> System.out.println(v),
	    e -> System.out.println("Error: " + e),
	    () -> System.out.println("Completed")
	);
```
##### takeUntil and skipUntil
Есть также два метода с названиями takeUntil и skipUntil. takeUntil работает точно так же, как takeWhile, за исключением того, что он берет элементы, пока предикат равен false. То же самое справедливо и для skipUntil.
Кроме того, takeUntil и skipUntil имеют очень интересную перегрузку. Точка отсечения определяется как момент, когда другая наблюдаемая испускает элемент.
