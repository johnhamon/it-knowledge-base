##### reduce
![[reduce.png]]
Саккумулирует все значения последовательности в одно, применив задаваемое в сигнатуре преобразование, пройдя по ней слева направо
```java
Observable<Integer> values = Observable.range(0,5);
		
values
	.reduce((i1, i2) -> i1 + i2)
	.subscribe(new PrintSubscriber("Sum"));
values
	.reduce((i1, i2) -> (i1 > i2) ? i2 : i1)
	.subscribe(new PrintSubscriber("Min"));
```
```
Sum: 10
Sum: Completed
Min: 0
Min: Completed
```

##### collect
Создание новой коллекции для каждого нового элемента неприемлемо. По этой причине Rx предлагает оператор `collect`, который делает то же самое, что и `reduce`, только на этот раз использует мутабельный аккумулятор. Используя `collect`, вы документируете, что не следуете соглашению о неизменяемости, а также немного упрощаете свой код:
```Java
Observable<Integer> values = Observable.range(10,5);
values
	.collect(
		() -> new ArrayList<Integer>(),
		(acc, value) -> acc.add(value))
	.subscribe(v -> System.out.println(v));
```
```
[10, 11, 12, 13, 14]
```

##### groupBy
Для каждого значения она вычисляет ключ и группирует значения в отдельные `Observable` на основе этого ключа.
![[groupby.png]]
Возвращаемое значение - `GroupedObservable`. Каждый раз, когда встречается новый ключ, будет испускаться новый внутренний `GroupedObservable`. Этот тип - не что иное, как стандартный `observable` с аксессором `getKey()` для получения ключа группы. По мере поступления значений из исходного `observable` они будут эмитироваться `observable` с соответствующим ключом.

Вложенные `Observable` могут усложнить сигнатуру, но их преимущество в том, что они позволяют группам начать испускать свои элементы до завершения работы исходной наблюдаемой.
