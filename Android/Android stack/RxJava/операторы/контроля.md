##### all
Устанавливает, что каждое значение, выдаваемое наблюдаемым, соответствует определенному критерию. Вот сигнатура и пример:
```java
public final Observable<java.lang.Boolean> all(Func1<? super T,java.lang.Boolean> predicate)
```
```java
Observable<Integer> values = Observable.create(o -> {
	o.onNext(0);
	o.onNext(10);
	o.onNext(10);
	o.onNext(2);
	o.onCompleted();
});


Subscription evenNumbers = values
	.all(i -> i % 2 == 0)
	.subscribe(
	    v -> System.out.println(v),
	    e -> System.out.println("Error: " + e),
	    () -> System.out.println("Completed")
	);
```
```
true
Completed
```
Интересным фактом в этом методе является то, что он возвращает наблюдаемую переменную с единственным значением, а не непосредственно булево значение. Это связано с тем, что неизвестно, сколько времени потребуется, чтобы определить, истинным или ложным должен быть результат. Даже если он завершает работу, как только узнает, это может занять столько же времени, сколько и сама исходная последовательность. Как только элемент не выполнит предикат, будет выдано значение `false`. Значение `true`, с другой стороны, не может быть выдано, пока исходная последовательность не завершится и не будут проверены _все_ элементы. Возврат решения внутри наблюдаемого объекта - удобный способ сделать операцию неблокируемой. В следующем примере мы видим, как `all` завершается как можно быстрее:
```java
Observable<Long> values = Observable.interval(150, TimeUnit.MILLISECONDS).take(5);
		
Subscription subscription = values
	.all(i -> i<3) // Will fail eventually
	.subscribe(
	    v -> System.out.println("All: " + v),
	    e -> System.out.println("All: Error: " + e),
	    () -> System.out.println("All: Completed")
	);
Subscription subscription2 = values
	.subscribe(
	    v -> System.out.println(v),
	    e -> System.out.println("Error: " + e),
	    () -> System.out.println("Completed")
	);
```
```
0
1
2
All: false
All: Completed
3
4
Completed
```
Если исходная наблюдаемая выдает ошибку, то все становится неважным, и ошибка проходит, завершая последовательность.
##### exists
Возвращает Observable, которое будет выдавать true, если любое из значений, выдаваемых Observable, делает предикат истинным.

##### isEmpty
Проверяет, эмитировал ли `Observable` значения перед завершением
##### contains
Метод contains устанавливает, эмитировался ли определенный элемент `Observable`. Использует метод `Object.equals` для установления равенства. Как и предыдущие операторы, он выдает свое решение, как только оно может быть установлено, и сразу же завершается.
![[contains.png]]
##### defaultIfEmpty
Если пустая последовательность вызовет у вас проблемы, вместо того чтобы проверять `isEmpty` и обрабатывать случай, вы можете заставить наблюдаемую выдать значение при завершении, если она ничего не выдала перед завершением.
![[defaultIfEmpty.png]]
