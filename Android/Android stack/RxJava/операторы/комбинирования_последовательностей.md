##### concat
![[concat.png]]
Состыковывает несколько последовательностей в одну. `concatWith` позволяет сцеплять их одну за одной в цепочке.
```Java
public void exampleConcatWith() {
	Observable<Integer> seq1 = Observable.range(0, 3);
	Observable<Integer> seq2 = Observable.range(10, 3);
	Observable<Integer> seq3 = Observable.just(20);
	
	seq1.concatWith(seq2)
		.concatWith(seq3)
		.subscribe(System.out::println);
}
```
##### repeat 
Позволяет сцеплять последовательность саму с собой конечное или бесконечное число раз. Не кэширует значения, чтобы повторять их. Когда придет время, он начнет новую подписку и избавится от старой.
##### repeatWhen
Если вам нужно больше контроля, чем дает repeat, вы можете контролировать момент начала повторения с помощью оператора `repeatWhen`. Время определяется `Observable`, которую вы предоставляете. Когда исходная последовательность завершается, она ждет, пока обрабатыващая `Observable` выдаст какое-либо сообщение (значение не имеет значения), и только после этого повторяется. Если обрабатыващая `Observable` завершается, это означает, что повторения должны прекратиться.

Может быть полезно, чтобы сигнал знал, когда повторение завершено. `repeatWhen` предоставляет специальную `Observable`, которая испускает void, когда повторение завершается. Вы можете использовать эту наблюдаемую для построения своего сигнала.
![[repeatWhen.png]]
```Java
public final Observable<T> repeatWhen(
    Func1<? super Observable<? extends java.lang.Void>,? extends Observable<?>> notificationHandler)
```

#### Конкурентные последовательности
##### amb
Принимает несколько `Observable` и возвращает то, которое первым эмитирует значение. Есть еще `ambWith`, чтобы удобно делать это в цепочке
![[amb.png]]
##### merge
Объединяет набор `Observable` в одну. Результирующая испускает значения, которые испускают исходные наблюдаемые, по мере того, как они их испускают. Это означает, что значения из разных последовательностей могут быть смешаны. Также есть `mergeWith` для цепочек
![[merge.png]]
##### mergeDelayError
Когда одна из последовательностей завершится с ошибкой, продолжит мержить остальные и выдаст ошибку только в конце
![[mergeDelayError.png]]
##### switchOnNext
Принимает `Observable`, которая эмитирует `Observable`-s. Возвращаемые элементы эмитируют свои элементы с самого недавнего `Observable`. Как только появляется новая, старая отбрасывается, и эмитируются значения из новой.
![[switchOnNext.png]]
##### switchMap
Если `flatMap` использует `merge` для объединения сгенерированных последовательностей, а `concatMap` - `concat`, то `switchMap` использует `switchOnNext` для фазы уплощения.
![[switchMap.png]]
#### Спаривание последовательностей
##### zip
![[zip.png]]
Берет две или более последовательностей и сопоставляет их значения один к одному по индексу. Для объединения значений требуется функция. В отличие от других сред, в RxJava zip по умолчанию не объединяет все значения в кортеже.  `zipWith` позволяет спаривать последовательности в цепочке.
##### combineLatest
Там, где zip использует индексы, `combineLatest` будет использовать время. Каждый раз, когда одна из объединяемых `Observable` выдает значение, это значение объединяется с последним значением другой наблюдаемой. И снова для объединения значений требуется функция.
![[combineLatest.png]]
