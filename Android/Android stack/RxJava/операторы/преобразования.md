##### map
Использует функцию преобразования, которая принимает элемент и возвращает новый элемент любого типа. Возвращаемый `Observable` состоит из значений, возвращенных функцией преобразования

##### cast и ofType
cast - это сокращение для преобразования приведения элементов к другому типу. Если у вас есть `Observable<Object>`, который, как вы знаете, будет эмитировать только значения типа T, то проще привести наблюдаемый объект, а не делать приведение в лямбда-функциях. Если в последовательности будет неприводимый тип, то метод выбросит исключение. Если не нужно исключение, используйте `ofType`, он отфильтрует элементы неприводимых типов.
##### timeStamp и timeInterval
Позволяют нам обогатить наши значения информацией об асинхронной природе последовательностей. `timestamp` преобразует значения в тип `Timestamped<T>`, который содержит исходное значение, а также временную метку того, когда было произведено событие.
```Java
Observable<Long> values = Observable.interval(100, TimeUnit.MILLISECONDS);

values.take(3)
	.timestamp()
	.subscribe(new PrintSubscriber("Timestamp"));
```
```Java
Timestamp: Timestamped(timestampMillis = 1428611094943, value = 0)
Timestamp: Timestamped(timestampMillis = 1428611095037, value = 1)
Timestamp: Timestamped(timestampMillis = 1428611095136, value = 2)
Timestamp: Completed
```
Временная метка позволяет нам увидеть, что элементы были выданы с интервалом примерно в 100 мс (Java дает мало гарантий на этот счет).

Если нас больше интересует, сколько времени прошло с момента последнего элемента, а не абсолютный момент времени, когда элементы были выданы, мы можем использовать метод `timeInterval`.
##### materialize and dematerialize
Также для логирования полезен `materialize`. `materialize` преобразует последовательность в представление метаданных.
```java
Observable<Long> values = Observable.interval(100, TimeUnit.MILLISECONDS);
		
values.take(3)
	.materialize()
	.subscribe(new PrintSubscriber("Materialize"));
```
```
Materialize: [rx.Notification@a4c802e9 OnNext 0]
Materialize: [rx.Notification@a4c802ea OnNext 1]
Materialize: [rx.Notification@a4c802eb OnNext 2]
Materialize: [rx.Notification@18d48ace OnCompleted]
Materialize: Completed
```
Тип `Notification` содержит методы для определения типа события, а также переносимого значения или `Throwable`, если таковые имеются. `dematerialize` отменяет действие `materialize`, возвращая материализованному наблюдаемому его обычную форму.
##### flatMap
`map` принимает одно значение и возвращает другое, заменяя элементы в последовательности один к одному. `flatMap` заменит элемент на любое количество элементов, включая нулевые или бесконечные. Метод преобразования `flatMap` принимает значения из исходного `Observable` и для каждого из них вернет новую `Observable`, которая эмитирует новые значения. Используется для уплощения вложенных коллекций. *Порядок следования элементов во входной последовательности не гарантирован в преобразованной*

##### concatMap
Обрабатывает последовательности последовательно, поэтому сохранит их порядок в возвращенной
##### flatMapIterable 
Принимает на вход `Iterable`, а не последовательности. Порядок не сохраняет
