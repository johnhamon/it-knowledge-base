Одна из ключевых особенностей Rx заключается в том, что вы не знаете, когда элементы будут испущены. Некоторые наблюдаемые объекты будут испускать все сразу и синхронно (например, `range`), некоторые испускаются через регулярные промежутки времени, а некоторые трудно или даже невозможно предсказать. Например, события мыши и UDP-пакеты просто приходят, когда приходят. Нам нужны инструменты, чтобы решать, что делать с этими событиями, не только на основе того, что они собой представляют, но и на основе того, когда они пришли и с какой частотой.
### Buffer
Позволяет собирать значения и получать их не по одному, а целыми пакетами. Существует несколько различных способов буферизации значений.
#### Полная, не перекрывающаяся буферизация
##### По числу элементов
Простейшая перегрузка сгруппирует фиксированное число значений вместе и испустит всю группу, когда она будет готова
![[buffercount.png]]
##### Периодическая
Следующая перегрузка позволяет буферизировать по времени. Время делится на окна одинаковой длины. Значения собираются для каждого окна, и в конце каждого окна буфер испускается.
![[buffer_timespan.png]]
##### По времени и числу элементов
![[buffer_timespancount.png]]
Буферизованные значения испускаются, если либо буфер заполнен, либо если временной интервал заканчивается и начинается новый.
##### С сигналом
Вместо периода времени вы также можете подавать сигнал буферу с `Observable` на очистку. При каждом сигнале `onNext` будут выводиться значения в буфере. Буферизация с помощью сигнала может быть очень полезна, если вы хотите придержать значения до того момента, когда вы будете готовы к ним.
![[buffer_signal.png]]


#### Перекрывающаяся буферизация
##### По числу элементов
![[buffer_countoverlap.png]]
Когда count > skip, буферы перекрываются.
Когда count < skip, элементы пропускаются.
Случай count = skip эквивалентен перегрузке без skip.
##### По времени
![[buffer_timespanshift.png]]
Когда timespan > timeshift, буферы накладываются друг на друга
Когда timespan < timeshift, элементы пропускаются.
Случай timespan = timeshift эквивалентен перегрузке без timeshift
##### По сигналу
Позволяет управлять началом и окончанием заполнения буфера, используя сигнальные `Observable`. 
```Java
public final <TOpening,TClosing> Observable<java.util.List<T>> buffer(
	Observable<? extends TOpening> bufferOpenings,
	Func1<? super TOpening,? extends Observable<? extends TClosing>> bufferClosingSelector)
```
![[buffer_openclose.png]]

Эта функция принимает два аргумента. Каждый раз, когда `bufferOpenings` выдает значение, начинается новый буфер. Вместе с открытием нового буфера значение, которое он испустил, передается в функцию `bufferClosingSelector`, которая использует значение для создания нового `Observable`, который будет сигнализировать о завершении соответствующего буфера, когда он испустит свое первое событие `onNext`

![[delay.png]]

##### delaySubscription
Можно также отсрочивать подписку на последовательность вместо хранения значений и испускания их позже, это более эффективно (если холодные `Observable`)
Можно откладывать и подписку, и эмиссию каждого элемента:
```Java
public final <U,V> Observable<T> delay(
	Func0<? extends Observable<U>> subscriptionDelay,
	Func1<? super T,? extends Observable<V>> itemDelay)
```
![[delay sub and each.png]]
Первый аргумент создает `Observable`, которая сигнализирует, когда подписываться. Вторая принимает на вход каждый элемент и задает, насколько его задержать.
