Холодные стримы (`Flow<T>`) отправляют значения только при вызове терминального оператора. До вызова и по окончании потока данных они не активны. Могут иметь только одного подписчика.
Так как билдер `flow {} `ничего не возвращает, сразу передает управление и отрабатывает, только когда на экземляре вызывается терминальный оператор - функции, возвращающие `Flow<T>` не suspend (не приостанавливаемые)

Горячие могут быть общими для нескольких подписчиков. Сборка значений из горячего потока не оповещает никакой producer-код (блок, который обновляет значение sharedFlow или stateFlow)
